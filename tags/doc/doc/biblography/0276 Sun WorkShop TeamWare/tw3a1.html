Access Control Tab

Related Topics:

Setting Global Permissions Setting Individual or Group Permissions

Use the Access Control tab to add, delete, and change access control permissions for the specified workspace. To open the Access Control tab, in the Configuring Window, choose Workspace Properties and select the Access Control tab.

This tab contains the following items:

Access Control Pane

Displays the current access control permissions for each operation performed on the specified workspace. You can grant or deny permission for the following Sun WorkShopTM TeamWare transactions::
bringover-from Bring over files from this workspace.
bringover-to Bring over files to this workspace.
putback-from Put back files from this workspace.
putback-to Put back files to this workspace.
undo Undo commands executed in this workspace.
workspace-delete Delete this workspace.
workspace-move Move this workspace.
workspace-reparent Reparent this workspace.
workspace-reparent-to Reparent other workspaces to this workspace.

This is how to determine the current access control settings. Transactions are listed in the left column, and permissions are listed in the right column.

  • If the right column is blank, all users have permission to execute that transaction.

  • If there is a minus sign on the line (-), then no users have permission to execute that transaction.

  • A person's login name in the right column indicates they have permission to perform the operation. If their name has a minus sign in front of it (-jdoe), then they are denied permission to perform that operation. An at-sign (@) before a name indicates it is a group.
Note - The order in which a user appears on the list can effect permissions. If a user is listed as having been both granted and denied permission, Sun WorkShopTM TeamWare tools only use the first reference. This can occur when you grant access to an individual and deny access to a group that the individual belongs to. (In this instance, if the individual is listed first, they will have access, if the group is listed first, the individual does not have access.)

Buttons
Load Reloads the information in the Access Control list.
Edit Opens the Access Control dialog box, where you can set new permissions.
