Setting a Default Workspace

Related Topics:

Loading Workspaces by Default

The CODEMGR_WS environment variable sets a workspace as the default workspace. A default workspace is loaded automatically when you start Configuring, Versioning, Merging and Freezepointing.

To set a default workspace:

setenv CODEMGR_WS /home/workspaces/myworkspace

This variable also makes the default workspace the default for command-line commands.When you use command-line commands (such as bringover, putback, freezept, or extract) the default workspace is automatically passed to the command (if you do not specify a workspace with the -w option).
