Reparenting A Workspace

Related Topics:

Creating a Child Workspace

Use reparenting to:

  • Reorganize a workspace hierarchy

  • Adopt an orphan workspace

You can also use reparenting to prepare to:

  • Populate a new top-level workspace

  • Move a feature into a new release

To give a child workspace a different parent:

1. In the Configuring Window, load the child workspace by choosing File Load Workspaces.

2. Select the workspace by clicking on it.

3. Choose Workspace Reparent.

4. In the Reparent Dialog Box, click on the browse button to select a parent workspace, or type the the name of the parent workspace.

If you do not specify a parent workspace, your workspace is orphaned. The workspace will still function, but it is no longer associated with a parent workspace.

5. Click OK.

The workspace shows a line connecting it to its new parent in the Configuring window.
