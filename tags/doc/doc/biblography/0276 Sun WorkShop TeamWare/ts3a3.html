Protecting Workspaces With Putback Validation

Use validation if you want to have more strict control over a workspace. If you turn on putback validation, only putbacks are allowed to the workspace. The user is also prompted for a "password" (Integration Request ID) before doing a putback

Note - Sun WorkShopTM TeamWare only records the Integration Request ID, it does not verify it. To check the Integration Request ID, you must write your own validation program. To do this, see Invoking Your Own Putback Validation Program.

To turn putback validation on:

1. In the Configuring Window, load an existing workspace by choosing File Load Workspaces.

2. Select the workspace by clicking on it.

3. Choose Workspace Properties.

4. Select the Putback Validation Tab in the Workspace Properties Dialog Box.

5. Click Putback Validation On.

6. Click OK.
