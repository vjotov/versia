Loading Workspaces by Default

Related Topics:

Setting a Default Workspace

You can use either environment variables or Tools Properties to load workspaces automatically.

Environment Variables

To load a workspace load automatically when Sun WorkShopTM TeamWare starts:
   Set the CODEMGR_WSPATH workspace environment variable to the location of the workspace. For example:

setenv CODEMGR_WSPATH /home/ws/myworkspace

To load more than one workspace:
   Put the list of workspaces inside double quotation marks. For example:

setenv CODEMGR_WSPATH "/home/ws/myworkspace /home/ws2/anotherws"

To load all the workspaces in one directory:
   Set the variable to a directory that contains multiple workspaces. For example:

setenv CODEMGR_WSPATH /home/workspaces/myworkspaces

Tools Properties

To load parent and child workspaces automatically:

1. In the Configuring Window, choose Options Configuring.

2. Select the Codemanager Options Tab in the Tools Properties Dialog Box.

3. Click the Load Workspaces with Parent and Children check box.
