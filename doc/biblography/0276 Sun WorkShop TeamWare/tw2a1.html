Bringover Create Tab

Related Topics:

Use the Bringover Create tab in the Transactions Dialog Box to a child workspace that includes files from the parent workspace. To open the Bringover Create tab, from the Configuring Window, choose Actions Bringover Create. You can also open this tab by dragging and dropping a workspace icon to an empty portion of the workspace pane.

This tab contains the following components:

File List Section
Parent Workspace (From) Directory text box Contains the absolute path of the parent directory.
To Child Workspace (To) Directory text box Type the absolute path of the child directory. If this directory does not already exist, Sun WorkShopTM TeamWare creates it for you.
Parent list Specifies which files and directories are included in the transaction:
Directories and Files Lets you select the files and directories you want included in the bringover.
File List Programs (FLPs) Lets you specify an FLP that generate lists of files for bringovers.
Entire Workspace Brings over all of the files and directories in the workspace.
List pane Resolves conflicts in files that are brought over from the parent workspace.

Buttons
Add Opens the Add Files Dialog Box, where you can select files to add to the List pane.
Load From args File Reloads the default arguments file list from the workspace args file.
Save To args File Saves the current list as the new default in the workspace args file.
Select All Selects all the files and directories that are listed in the List pane.
Deselect All Deselects all the files and directories that are listed in the List pane.
Delete Removes the selected file or directory from the list.

Options Check Boxes
Preview Previews by running a bringover without actually transferring files.
Verbose Increases the information displayed in the Transaction Output dialog box. By default, a message is displayed for each created, updated, or conflicting file. The Verbose option causes Configuring to print a message for all files, including those that are not brought over.
Quiet Suppresses the output of status messages to the Transaction Output dialog box.
Delta Comments Provides delta number, owner and comments in the Transaction Output dialog box.
Skip SCCS gets This option prevents extraction of g-files as a part of a bringover. Using this option improves file transfer performance, although it shifts the responsibility to you to do the appropriate file maintenance (with SCCS get command).
Force Conflicts Causes all updates to be treated as conflicts.
Skip Backups Skips the step of copying the existing files to the Codemgr_wsdata/backup/files directory in the destination workspace. This option reduces the disk space occupied by the child workspace and improves file transfer performance, but it removes the option of using Undo.

Comments Pane

Type your comments about the bringover here to have them stored in the workspace history log (Codemgr_wsdata/history) in the child workspace.

Buttons
Bringover Starts the bringover.
Show Output Opens the Transaction Output dialog box.
Close Closes the Transaction dialog box.
Help Displays online help for this tab.
