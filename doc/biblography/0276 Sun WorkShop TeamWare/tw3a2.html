Protecting Workspaces With Putback Validation

Use the Putback Validation tab to set up putback validation to protect a workspace. To open the Putback Validation tab, from Configuring, choose Workspace Properties and select the Putback Validation tab.

This tab contains the following items:

Buttons
Putback Validation radio buttons Lets you turn putback validation on or off. When Putback Validation is on, only putbacks to the workspace are allowed and the user is prompted for an "Integration Request ID."
Override Access Control radio buttons Lets you override access control. "Yes" gives putback validation access to workspaces even if it is not set in the Access Control dialog box.
Putback Approval radio buttons By default, Sun WorkShopTM TeamWare allows all putbacks to the workspace. The Integration Request ID is not validated by TeamWare. If you want to have your own program validate the Integration Request ID, select Approved by Validation Program, and type the program name in the Validation Program text box. See Invoking Your Own Putback Validation Program
Validation Program text box Indicate the full path name of your validation program, for example: /home/foster/scripts/valid1.
