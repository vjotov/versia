Codemanager Options Tab

Use the Codemanger Options tab to set options for Configuring, such as double-click action, whether to load parent and children workspaces by default, and how workspaces are displayed. To open the Configuring Options tab, in the Configuring Window, choose Options Configuring and select the Codemanager tab.

This tab contains the following items:
Working Directory text box Type the name of the directory where you want to automatically load workspaces when the program starts.
Workspace Double Click Action radio buttons Select what you want to happen when you double click on a workspace: Versioning, History Browser or Other. If you select Other, type the path name of the command in the text box.
Conflict Workspace Double Click Action radio buttons Select what you want to happen when you double click on a conflicted workspace: Resolve Pane or Other. If you select Other, type the path name of the command in the box.
Load Workspaces check box Click to automatically load the parent and child workspace(s) of a workspace when it is loaded.
Load Children radio buttons Select how you want to load child workspaces:
Selective Brings up a dialog box for you to select the child workspaces you want to load.
All Loads all child workspaces.
Orientation radio buttons Select the layout of the workspace hierarchy in the Workspace pane:
Vertical Sets vertical layout with parent at the top and children below.
Horizontal Sets horizontal layout with parent to the left and children branching to the right.
Workspace Names radio buttons Select the display of workspace names in the Workspace pane:
Full Displays the full path name of the workspace.
Short Displays the working directory name of the workspace.
Descriptive Displays the descriptive name for the workspace. See Giving a Workspace a Descriptive Name.
