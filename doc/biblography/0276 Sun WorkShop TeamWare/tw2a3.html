Putback Tab

Use the Putback tab in the Transactions Dialog Box to copy files back to a parent workspace. To open the Putback tab, from the Configuring Window, choose Actions Putback. You can also initiate a Putback transaction by dragging and dropping a child workspace icon over its parent's icon.

In this tab are the following components:

File List Section
Parent Workspace (From) Directory text box Contains the absolute path name of the parent directory
Child Workspace (To) Directory text box Contains the path name of the child directory.
Parent Files Specifies which files and directories will be involved in the transaction:
Directories and Files radio button Lets you select the files and directories you want included in the bringover.
File List Programs (FLPs) radio button Lets you specify a FLP that generate lists of files for bringovers.
Entire Workspace radio button Uses all of the files and directories in the workspace.
List pane Lists files and directories.

Buttons
Add Opens the Add Files Dialog Box, where you can select files to add to the List pane.
Load From args File Reloads the default arguments file list from the workspace args file.
Save To args File Saves the current list as the new default in the workspace args file.
Select All Selects all the files and directories listed in the List pane.
Deselect All Deselects all the files and directories listed in the List pane.

Options Check Boxes
Preview Previews by running a bringover without actually transferring files.
Verbose Increases the information displayed in the Transaction Output dialog box. By default, a message is displayed for each created, updated, or conflicting file. The Verbose option causes Configuring to print a message for all files, including those that are not brought over.
Quiet Suppresses the output of status messages to the Transaction Output dialog box.
Delta Comments Provides additional information in the Transaction Output dialog box: delta number, owner and comments.
Skip SCCS gets Inhibits the invocation of the SCCS get program as part of the putback. Normally, g-files are extracted after they are brought over. This option improves file transfer performance, although it shifts the responsibility to you to do the appropriate gets.
Force Conflicts Causes all updates to be treated as conflicts.
Skip Backups Skips the step of copying the existing files to the Codemgr_wsdata/backup/files directory in the destination workspace. This option reduces the disk space occupied by the child workspace and improves file transfer performance, but it removes the option of using Undo.

Comments Pane

Type your comments about the putback here to have them stored in the workspace history log (Codemgr_wsdata/history) in the child workspace. See Viewing and Customizing Workspace History.

Buttons
Putback Starts the putback.
Show Output Opens the Transaction Output dialog box.
Close Closes the Transaction dialog box.
Help Displays online help for this tab.
